# Project overview
- Purpose: Explore Reversi (6x6) reachability from the starting position under placement masks, and synthesize a move progression if the target board is reachable. Uses SMT (Yices2 delegated to Kissat) to model game rules and search for a satisfying move sequence.
- Tech stack: Rust 1.91 (rustfmt, clippy), dependencies include easy-smt (SMT driver), rand, rayon, itertools, derive-getters, num_cpus. External solver binary `yices-smt2` with `--delegate=kissat` must be available.
- Entry points: `src/main.rs` builds a random target board consistent with placement masks, prints it, runs the SMT solver, and prints a found progression; `src/bin/randomly_generate_game.rs` samples random full-length game records.
- Domain model: `board.rs` implements Board (6x6 grid), placement masks, move legality/flip logic, conversions to/from bitboards and octal strings; `game.rs` defines initial position, move records (`UncheckedGameProgression`), serialization (`A1..F6` pairs) and simulation; `reachability_problem.rs` defines the reachability instance and solver trait/result enum; `yices2_kissat_reachability_solver.rs` encodes the game as bitvector SMT with constraints over per-move bitboards, pass handling, and flip computation.
- Data formats: Boards/placement masks may be encoded as 12-digit octal strings (3 bits per cell chunked by columns/rows). Move records are concatenated `A-F` + `1-6` pairs (e.g., `C3D3...`).
- Behavior: Black starts; per-turn pass allowed only if no legal moves. Solver reconstructs move positions from model bitvectors and returns `UncheckedGameProgression`.
